ICAREdata® Structured Phrases to FHIR Mapping Guidance

There are currently two structured phrases covering disease status, which is an mCODE™ data element (see mCODE™ CancerDiseaseStatus profile) and treatment change, which is covered in this implementation guide, as it is not part of the mCODE™ specification.

Cancer Disease Status

The ICAREdata® Cancer Disease Status structured phrase maps the mCODE™ FHIR profile by the same name. Figure 1 shows the detailed mapping of this ICAREdata structured phrase to specifics of the mCODE™ FHIR profiles and value sets.

Each element of the structured phrase is mapped to mCODE™ as follows:

  • Lesion evaluated:
    • Primary tumor maps to CancerDiseaseStatus.focus, which in turn references the PrimaryCancerCondition profile.
    • Metastatic lesion maps to CancerDiseaseStatus.focus, which in turn references the SecondaryCancerCondition profile.
    • Not evaluated maps to CancerDiseaseStatus.DataAbsentReason.valueCodeableconcept (to the value set member 'not asked'.
  • Status value: maps to CancerDiseaseStatus.valueCodeableConcept, with the following vocabulary mappings:

    • ICAREdata® status value mCODE™ ConditionStatusTrendVS value set
    Code Code Description
    No evidence of disease 260415000 In full remission. Use this term for expressing NED (no evidence of
    disease), condition resolved, or full remission.
    Responding 385633008 Improving
    Stable 58158008 Stable
    Progressing 230993007 Worsening

  • Reason value maps to CancerDiseaseStatus.Extensions.obf-evidenceType-extension.valueCodeableConcept, with the following vocabulary mappings:
